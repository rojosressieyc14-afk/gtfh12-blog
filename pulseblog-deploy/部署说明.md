# PulseBlog 宝塔部署指南

## 文件说明

```
pulseblog-deploy/
├── PulseBlog/           # 前端静态文件（上传到网站根目录）
│   ├── index.html       # 博客前台
│   ├── assets/          # 前台资源
│   └── admin/           # 后台管理
│       ├── index.html
│       └── assets/
├── pulseblog-api        # Go 后端程序（Linux amd64）
├── pulseblog.service    # 系统服务配置
└── 部署说明.md           # 本文件
```

---

## 第一步：宝塔创建网站

1. 登录宝塔面板
2. 左侧 → **网站** → **添加站点**
3. 填写：
   - **域名**：`www.kangban.online`
   - **根目录**：默认即可（通常是 `/www/wwwroot/www.kangban.online`）
   - **PHP版本**：选"纯静态"
   - 点击**提交**

---

## 第二步：上传前端文件

1. 在宝塔 → 网站 → 点击 `www.kangban.online` 的**根目录**按钮
2. 把 `pulseblog-deploy/PulseBlog/` **整个文件夹**上传进去
3. 最终路径应该是：
   ```
   /www/wwwroot/www.kangban.online/PulseBlog/
       ├── index.html
       ├── assets/
       └── admin/
   ```

验证：浏览器打开 `http://www.kangban.online/PulseBlog/`，应该能看到页面（接口暂时连不上是正常的）。

---

## 第三步：配置反向代理

1. 宝塔 → 网站 → 点击 `www.kangban.online` 的**设置**
2. 左侧 → **反向代理** → **添加反向代理**
3. 填写：
   - **代理名称**：`PulseBlog API`
   - **目标URL**：`http://127.0.0.1:8080`
4. 点击**提交**
5. 然后点击反向代理列表右侧的**配置文件**按钮
6. **删除默认内容**，替换为以下配置：

```nginx
# API
location /PulseBlog/api/ {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 60s;
}

# 上传文件
location /PulseBlog/uploads/ {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    expires 7d;
    add_header Cache-Control "public";
}
```

7. 点击**保存**

---

## 第四步：配置伪静态（SPA 路由支持）

1. 宝塔 → 网站设置 → **伪静态**
2. 选择 `thinkphp`（或者选 `其他`）
3. 粘贴以下内容：

```nginx
location /PulseBlog/ {
    try_files $uri $uri/ /PulseBlog/index.html;
}

location /PulseBlog/admin/ {
    try_files $uri $uri/ /PulseBlog/admin/index.html;
}
```

4. 点击**保存**

> 注：如果之前已经通过"反向代理"配置了上述内容，伪静态这里可以跳过。

---

## 第五步：申请 SSL 证书（免费 HTTPS）

1. 宝塔 → 网站 → 点击 `www.kangban.online` 的**设置**
2. 左侧 → **SSL**
3. 选择 **Let's Encrypt**
4. 勾选域名 `www.kangban.online`
5. 点击**申请**
6. 申请成功后，打开 **强制HTTPS** 开关

至此，你的网站就支持 HTTPS 了，浏览器自动跳转。

---

## 第六步：上传并运行后端

### 6.1 创建后端目录

SSH 连接服务器，执行：

```bash
mkdir -p /www/server/pulseblog/uploads
```

### 6.2 上传后端程序

通过宝塔的文件管理器：
1. 进入 `/www/server/pulseblog/`
2. 上传 `pulseblog-api` 文件
3. 右键 → **权限** → 设为 `755`（或 `0755`）
4. 上传完成后执行：

```bash
chmod +x /www/server/pulseblog/pulseblog-api
```

### 6.3 配置系统服务

1. 在宝塔文件管理器，进入 `/etc/systemd/system/`
2. 上传 `pulseblog.service`
3. 点击编辑，修改以下几项（替换成你的真实值）：

```
DB_PASSWORD=你的MySQL密码
JWT_SECRET=用下面的命令生成
DEFAULT_ADMIN_PASSWORD=你的管理员密码
```

生成 JWT_SECRET 的命令：

```bash
openssl rand -hex 32
```

5. 保存后执行：

```bash
# 重新加载服务
systemctl daemon-reload

# 启动服务
systemctl start pulseblog

# 设置为开机自启
systemctl enable pulseblog

# 查看运行状态
systemctl status pulseblog
```

看到 `active (running)` 就说明后端跑起来了。

## 常见问题

**Q：打开页面空白 / 404？**
→ 检查伪静态配置是否正确，确保 `try_files` 指向了正确的路径。

**Q：接口报 502 Bad Gateway？**
→ 后端没启动。SSH 执行 `systemctl status pulseblog` 查看错误日志。

**Q：登录报错？**
→ 检查 `pulseblog.service` 中的数据库配置是否正确，MySQL 是否已启动且数据库存在。

**Q：上传图片不显示？**
→ 检查反向代理中 `/PulseBlog/uploads/` 的配置是否生效。
